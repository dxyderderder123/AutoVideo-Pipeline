## ⚙️ 项目架构与工作流程

本项目实现了从**零散文章到成品发布**的全链路自动化。它不是简单的脚本堆砌，而是一个具备硬件感知（GPU锁）、API熔断降级和多线程并行能力的工业级管线。

### 1. 核心管线逻辑 (Pipeline Steps)

系统将视频制作拆解为 9 个独立步骤，由 `src_english/workflow.py` 统筹调度：

| 步骤 | 功能描述 | 核心技术/模型 |
| :--- | :--- | :--- |
| **01 分析** | 语义切构与关键词提取 | LLM (DeepSeek/GPT) |
| **02 配音** | 高拟真长文语音合成 | **VibeVoice** (含自动权重下载) |
| **03 素材** | 视频素材抓取与配额熔断 | Pexels/Pixabay API + 缓存池 |
| **04 对齐** | 音频字幕毫秒级强制对齐 | WhisperX / 词频比例算法 |
| **05 字幕** | 视觉短句边界切分 | 动态文本折行算法 |
| **06 翻译** | 双语字幕生成与ASS渲染 | LLM 翻译 + libass 滤镜 |
| **07 混剪** | 视音频底层物理合成 | **FFmpeg** 滤镜链 (scale/crop/concat) |
| **08 封面** | AI 艺术封面生成 | SiliconCloud (Kolors/Flux) |
| **09 发布** | B站全自动风控对抗上传 | bilibili-api + WAF 指纹绕过 |

### 2. 批量处理与运行 (Usage)

为了适配大规模产出需求，本项目支持**一键扫描式批量生产**：

- **自动化入口**：只需将多篇 Markdown 格式的文章放入 `workspace/input/`。
- **1-Click 运行**：双击根目录或 `scripts/` 下的 `run_batch.bat` 即可启动。
- **并行调度**：系统会自动检测 GPU 状态，根据显存大小决定并行任务数（防止爆显存），并顺序调用 `main.py` 完成所有视频的渲染与上传。

## 📈 开发动机与技术演进：从“过度工程”到“务实原生”

本项目的诞生并非一蹴而就，而是一个经历过“高大上框架挫败”后，向底层原生技术回归的过程：

1.  **初衷 (一年前)**：最初我采用半自动化的方式制作科普视频，画面多为静态配图，效率低下且视觉表现力有限。
2.  **转折 (初试结束后)**：为了追求工业级的动态效果，我曾尝试引入 **Remotion** (基于 React 的视频框架) 期望实现复杂的前端动画，并试图集成 **Manim** (数学动画引擎) 以精准支撑理科知识的演示。
3.  **教训与反思**：实际开发中发现，这些顶层框架虽然华丽，但在处理长维度的自媒体内容时，渲染成本极高且稳定性极差。由于环境依赖过于复杂，生成的视频质量在未经过度调优前完全无法达到商用标准，导致了数周的进度停滞。
4.  **最终落地**：在反复试错中，我意识到“伟大的架构不在于堆叠框架，而在于解决底层边沿的缺陷”。我将目光转向最底层的 **FFmpeg** 与原生 Python 逻辑。通过解耦原本复杂的系统，我意外地（或作为研究过程中的副产品）落地了当前这个**全自动英语视频工作流**。它极大地提升了稳定性，实现了 100% 的生产自动化。
