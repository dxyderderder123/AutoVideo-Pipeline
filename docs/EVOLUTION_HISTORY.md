# 英语自动化项目：开发与优化历程

这个文档记录了我在开发这个“英语视频全自动生成”项目时，遇到的一些关键技术问题和我是怎么解决的。

## 1. 解决视频“镜头太死”的问题
*   **问题**：一开始，如果一句英文很长（比如20秒），那么画面就会一直停留在同一个镜头上，看起来非常枯燥。
*   **解决**：我修改了代码逻辑，让程序每隔几秒就自动找一个新的视频画面，然后把这些小片段拼在一起，配合那一长句配音。这样即使一句话很长，画面也是动态变化的。

## 2. 解决字幕“对不准”和“满屏都是字”
*   **问题**：之前字幕总是对不齐，声音还没完字幕就没了，或者字幕在屏幕上堆成一坨，非常难看。
*   **解决**：
    *   **精准对齐**：我重新设计了时间戳算法，确保每一毫秒都对得准，消除了之前的同步误差。
    *   **自动切短**：现在程序会自动检测字幕长度，如果句子太长，它会自动把字幕切成两三行显示，并且每行只显示几秒钟，看起来清爽多了。

## 3. 解决 B 站上传被拦的问题
*   **问题**：自动上传的时候，B 站总报错 `406`，说我上传太快，其实是把它当成机器人了。
*   **解决**：我研究了 B 站的拦截机制，手动把真实的浏览器“指纹”（也就是 Cookie 里的核心信息）补进了程序里。这样程序在发稿时，B 站会以为是一个真人在操作，大大提高了自动发布的成功率。

## 4. 解决 API 费用太贵的问题
*   **问题**：如果视频素材全部从网上买/下，API 额度很快就用完了。
*   **解决**：我加了一个“省钱逻辑”。当同一个视频素材已经下过，或者当日下载量太大时，程序会自动从本地的“老素材库”里挑选合适的画面来复用。这样既保证了视频质量，又帮我省了不少钱。
