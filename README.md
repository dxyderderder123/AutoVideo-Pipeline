## ⚙️ 项目架构与工业化工作流 (Project Architecture & Workflow)

本项目并非单次运行的小脚本，而是一套支持**全自动批量化生产**的工业级管线。它能够自动扫描并处理 `workspace/input/` 文件夹下的所有文章，实现从“选题录入”到“成品产出”的流水线作业。

### 🚀 运行入口
*   **Windows 极速启动**：双击运行根目录下的 `scripts/run_batch.bat` (或 `scripts/batch_run.py`)。它会自动唤起内置虚拟环境，调用并发调度器进行批量出片。
*   **单篇调试**：
    ```bash
    # 针对单篇文章进行精准调试或极速预览
    python main.py --input "3中高阶.md" --preview
    ```

### 🛠️ 核心管线解耦 (Core Pipeline)
系统通过 `workflow.py` 状态机驱动，将任务拆分为 9 步原子化操作，确保每一步都具备异常重试与状态缓存能力：

| 步骤 | 模块 | 技术要点 |
|---|---|---|
| **01** | `step1_analyze` | **语义解耦**：利用 LLM 将长文本按人类语义习惯切分为 20-30s 的逻辑段落，并提取视觉关键词。 |
| **02** | `step2_tts` | **克隆播报**：调用 VibeVoice 生成高拟真人声。具备 **GPU 显存锁**，支持多进程调度下的资源排队。 |
| **03** | `step3_video` | **智能爬虫/熔断**：动态抓取 Pexels/Pixabay 视频。内置权重随机算法，优先复用本地素材以节省 API 配额。 |
| **05** | `step5_subtitle` | **毫秒级对齐**：基于语音包实际时长，通过词频权重算法将长配音快速切分为更具视觉冲击力的短句字幕。 |
| **06** | `step6_translate` | **双语渲染**：LLM 意译 + ASS 特效字幕生成。解决了开源方案常见的“时间轴累积漂移”硬伤。 |
| **07** | `step7_merge` | **FFmpeg 渲染器**：底层 C 级媒体流合成。支持 720p/1080p 动态缩放、镜头裁切与多分辨率自适应。 |
| **08** | `step8_cover` | **视觉包装**：基于文章主题，由 SiliconCloud Kolors 引擎生成对应的 16:9 横版/ 9:16 竖版高清封面。 |
| **09** | `step9_upload` | **自动推流**：过 Bilibili WAF 的风控伪装上传。支持定时发布与视频属性自动标。 |

### 📂 自动化生产保障 (Safety & Reliability)
*   **硬件调度** (`utils_hardware`)：实时监测 GPU 温控与显存，防止在并发压测下导致显存溢出。
*   **断点续传**：每一篇生成的视频都在 `workspace/output/YYMMDD/` 下拥有独立的工程目录，失败的任务可在修复后直接复用已完成的步骤资产。
*   **质量抽检**：批量任务结束后，系统会播放 `finish.mp3` 提示，并在 `logs/` 生成耗时与 API 消耗分析报告。
