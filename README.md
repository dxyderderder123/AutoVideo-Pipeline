## ⚙️ 项目原理简介 (Pipeline Overview)

本项目核心采用了**原生 API 级调用与底层 FFmpeg 滤镜链混剪**，分为以下核心阶段：

1. **智能意群切分与资产匹配 (Text Analysis & Asset Scraping)**
   - 利用大语言模型将长文本进行符合人类阅读顿挫的“意群切割”，并提取视频检索关键词。
   - 内置了 **API配额保护器（Quota Protection）**：爬虫模块（`step3_video`）自带熔断降级算法。一旦超出预算请求，立即降级使用本地历史“视频池”，有效防止云端 API 恶性循环并耗尽配额。

2. **音频聚合合成与微调对齐 (TTS & Time Alignment)**
   - 针对长片段和短断句截然不同的声学特性，设计了动静结合的传参：给 TTS 层喂入整段长文本（避免切碎导致的引擎幻觉和 BGM 底噪杂音），在后续生成阶段再依据标点拆分。

3. **FFmpeg 动态底图混剪与防漂移 (Rendering & Alignment)**
   - 弃用不稳定的前端渲染工具，所有转场与组装全部下沉至 `FFmpeg` 脚本链完成。
   - 解决了困扰开源社区的字幕声音漂移难题：统一内部时钟源，在 ASS 字幕中植入 `\N` 强制换行并在代码层剥离多余强制等待延迟，做到多动态短分镜中，长短句依然完美卡点输出、永不漂移。
