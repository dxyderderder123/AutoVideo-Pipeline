## ⚙️ 项目架构与工作流程

本项目实现了从**输入文章到视频成品**的全链路自动化逻辑。系统具备硬件感知（GPU锁）、API熔断降级和多线程并行能力，确保了生产环境的稳健性。

### 1. 核心自动化管线 (Pipeline)

系统将视频制作逻辑拆解为 9 个独立步骤，并由核心逻辑统筹调度：

| 阶段 | 功能描述 | 核心技术 |
| :--- | :--- | :--- |
| **01 分析** | 语义切片与关键词提取 | LLM (NLP 语境分析) |
| **02 配音** | 高拟真语音合成 | **VibeVoice** 推理引擎 |
| **03 素材** | 视觉素材抓取与熔断保护 | Pexels/Pixabay API 接入 |
| **04 对齐** | 音频与文本毫秒级强制对齐 | WhisperX / 词频比例算法 |
| **05 字幕** | 视觉短句边界切分 | 动态文本折行逻辑 |
| **06 翻译** | 双语字幕生成与渲染 | LLM 翻译 + libass 渲染 |
| **07 混剪** | 视音频底层物理合成 | **FFmpeg** 媒体处理滤镜链 |
| **08 封面** | AI 艺术封面生成 | 扩散模型 (Diffusion Model) |
| **09 发布** | 自动化分发与上传 | 协议指纹模拟 (WAF 对抗) |

### 2. 批量生产能力

本项目支持**一键扫描式批量生产**：

- **自动化入口**：系统自动扫描 `workspace/input/` 目录下的所有 Markdown 文稿。
- **并行调度**：自动检测 GPU 显存负载，动态决定并行任务数，在保证硬件安全的前提下最大化生产效率。
- **一键运行**：通过 `scripts/run_batch.bat` 即可启动完整的流水线任务。

## 📈 技术演进历程：从实战中锤炼的工程直觉

本项目的核心逻辑并非一蹴而就，而是在过去一年备考压力与实战需求的博弈中，不断自我否定的结果。

### 第一阶段：半自动化的初期探索 (Early 2024)
*   **初衷**：为了在备考初试期间实现经济独立，我开始探索科普视频制作。由于精力需高度集中于物理专业课复习，我必须尽可能压缩体力劳动成本。
*   **实现**：由于当时 AI 对精确数学逻辑（物理公式）的生成能力尚弱，我敏锐地判断了技术边界，转向更易发挥 AI 视觉优势的“哲学科普”。我搭建了一套基于 `ComfyUI` (生图) + `GPT-SoVITS` (语音克隆) 的半自动流。
*   **历史痕迹**：[archive/old_workflow/素材生成_双人对话.py](file:///v:/Default/Desktop/Self-media/archive/old_workflow/素材生成_双人对话.py)。这标志着我早期已经实现了“文稿输入 -> 分镜脚本 -> 自动化素材生成”的部分闭环。

### 第二阶段：高层框架的理想与幻灭 (Post-Preliminary Exam)
*   **跨越**：初试结束后，我偶然在官网了解到王磊老师在“物理与 AI 交叉领域”的研究方向，产生了强烈的技术共鸣。我试图将这一年积累的 AI 经验从“应用层”转向“系统层”探索。

*   **反思**：经过一个月的死磕，我识别出了高层框架的局限性——它们在全自动流水线中渲染成本过高、稳定性差、且对几何关系的控制极其复杂，最终产出的成品仅有“动态 PPT”的低质感。

*   **决断**：面对复试阶段（四大力学、普通物理）的繁重复习压力，我意识到“伟大的架构在于解决底层边沿的缺陷”。我果断止损，放弃高层框架，重新构建了这套使用最原生 **FFmpeg 滤镜链**驱动的稳健流水线。
*   **成果**：这套方案不但通过了工业级的生产测试，更展现了我的工程逻辑——即在逆境与压力下，依然能够通过**解耦系统、控制 API 熔断、优化硬件调度**来交付成品。
